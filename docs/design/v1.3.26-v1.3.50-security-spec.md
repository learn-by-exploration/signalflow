# SignalFlow AI — Security & Testing Iterations v1.3.26–v1.3.50

> **Spec for 25 security iterations, each a standalone test file committed and tagged.**
> Covers all gaps identified after v1.3.1–v1.3.25 security sweep.

---

## Overview

| Range | Theme |
|-------|-------|
| v1.3.26–v1.3.30 | WebSocket, JWT deep, payment/webhook security |
| v1.3.31–v1.3.35 | Celery, Telegram, AI prompt injection, cache, timing |
| v1.3.36–v1.3.40 | Admin, tier bypass, cross-tenant, log safety, password hardening |
| v1.3.41–v1.3.45 | Signal integrity, portfolio manipulation, schema enforcement, DB constraints, HTTP method |
| v1.3.46–v1.3.50 | Error hardening, config safety, news injection, rate limit edge, regression + final audit |

### Conventions

- **Test framework**: pytest with `asyncio_mode=auto`
- **Primary fixture**: `test_client` — `AsyncClient` with auth overrides (`user_id=00000000-0000-0000-0000-000000000099`, `tier=pro`, `telegram_chat_id=12345`)
- **Secondary fixture**: `web_user_client` — web-only user (`telegram_chat_id=None`, `tier=free`)
- **API prefix**: `/api/v1/`
- **All tests mock external services** (Claude API, Binance, Razorpay, Telegram, Redis where necessary)
- **No destructive operations** — tests use in-memory SQLite via `seeded_db` fixture
- **File naming**: `test_v1326_websocket_security.py` through `test_v1350_final_audit.py`

## Tagging & Commit Protocol

Each iteration follows:

```bash
# 1. Write test file
# 2. Run full test suite (all tests must pass)
cd backend && python -m pytest tests/ -v --override-ini="asyncio_mode=auto"

# 3. Commit
git add backend/tests/test_v13XX_*.py
git commit -m "test: v1.3.XX — <title>"

# 4. Tag
git tag -a v1.3.XX -m "v1.3.XX — <title>"
```

Tests that expose real vulnerabilities requiring code fixes will:
1. Be committed first with the vulnerability flagged (test may `pytest.skip` or `xfail`)
2. Fix committed separately: `fix: <description>`
3. Test updated to assert the fix, then re-tagged
