### Task 1.6: Fix Celery Singleton Engine
**Priority**: P1 | **Effort**: 3h | **Risk**: Medium  
**Architect amendment**: Add cleanup signal, thread-safe double-check lock, pool_size=10.

**Create**: `backend/app/tasks/_engine.py`
```python
import threading
from celery.signals import worker_process_shutdown

_engine = None
_session_factory = None
_lock = threading.Lock()

def get_task_engine():
    global _engine
    if _engine is None:
        with _lock:
            if _engine is None:  # double-check
                _engine = create_async_engine(
                    settings.database_url,
                    pool_size=10, max_overflow=5,
                    pool_pre_ping=True, pool_recycle=1800,
                )
    return _engine

@worker_process_shutdown.connect
def dispose_engine(**kwargs):
    global _engine
    if _engine:
        import asyncio
        asyncio.get_event_loop().run_until_complete(_engine.dispose())
        _engine = None
```

### Task 1.14: JWT Revocation Store *(Moved from Phase 2 per Security review)*
**Priority**: P1 | **Effort**: 5h | **Risk**: Medium  
**Security amendment**: Use per-user `iat` check for bulk revocation, fail closed on Redis failure.

**Implementation**:

1. **Per-token revocation** — Redis: `token_bl:{jti}` with TTL=1800s (30 min)
2. **Bulk revocation** — Redis: `user:{id}:tokens_invalid_before` = timestamp
3. **Check in `require_auth()`** (not `decode_jwt_token()`):
```python
async def require_auth(...):
    payload = decode_jwt_token(token)
    # Check individual blacklist
    if await redis.exists(f"token_bl:{payload['jti']}"):
        raise HTTPException(401, "Token revoked")
    # Check bulk invalidation
    invalid_before = await redis.get(f"user:{payload['sub']}:tokens_invalid_before")
    if invalid_before and payload['iat'] < float(invalid_before):
        raise HTTPException(401, "Session expired")
    return AuthContext(...)
```
4. **Fail closed**: If Redis unreachable during blacklist check → reject token (security over availability)
5. **Triggers**: logout, password change, account delete, logout-all

### WebSocket Redis PubSub Sub-Plan *(Expanded per Architect review)*

**Channel design**: One channel per market type: `ws:stock`, `ws:crypto`, `ws:forex`, `ws:all`
**Subscription**: Each uvicorn process subscribes to all channels. Per-client market subscriptions filter locally.
**Publisher**: Signal generation task publishes to relevant channel: `redis.publish("ws:stock", signal_json)`
**Backpressure**: Per-client bounded message queue (max 50 messages). Drop oldest if full.
**Reconnection**: Redis PubSub reconnect with exponential backoff (1s, 2s, 4s, max 30s)
**Serialization**: Serialize once in publisher (JSON bytes). Subscribers decode + fan out to local clients.
