### Task 1.6: Fix Celery Singleton Engine
**Priority**: P1 | **Effort**: 2h | **Risk**: Medium (affects task execution)  
**Expert consensus**: Architect — connection pool thrash

**Problem**: `signal_tasks.py` creates a new `create_async_engine()` on every task invocation (every 5 min × 2 task types = 24 engines/hour, each opening/closing DB connections).

**File to create**: `backend/app/tasks/_engine.py`
```python
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession, async_sessionmaker
from app.config import get_settings

_engine = None
_session_factory = None

def get_task_engine():
    global _engine
    if _engine is None:
        settings = get_settings()
        _engine = create_async_engine(
            settings.database_url,
            pool_size=5,
            max_overflow=5,
            pool_pre_ping=True,
            pool_recycle=1800,
        )
    return _engine

def get_task_session_factory():
    global _session_factory
    if _session_factory is None:
        _session_factory = async_sessionmaker(get_task_engine(), expire_on_commit=False)
    return _session_factory
```

**File to modify**: `backend/app/tasks/signal_tasks.py` lines 26, 36, 53
- Replace `engine = create_async_engine(...)` with `engine = get_task_engine()`
- Remove `await engine.dispose()` calls
