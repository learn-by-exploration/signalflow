## Next-Phase Roadmap

### Guiding Principle

> **Stop building features. Fix the foundation. Get users. Prove signals work.**
> — Consensus from PM, Investor, Data Scientist

### Phase 1: FOUNDATION FIX (Weeks 1-3) — P0 Blockers

These items must be completed before any public launch. Each blocks a critical path.

#### 1.1 Fix Web User Identity (Week 1)
- **What**: Update all user-scoped queries to use `user_id` alongside `telegram_chat_id`
- **Files**: `portfolio.py`, `price_alerts.py`, `alerts.py`, all endpoints querying by `telegram_chat_id`
- **Pattern**: `or_(Model.user_id == auth.user_id, Model.telegram_chat_id == auth.telegram_chat_id)`
- **Test**: Add `web_user_client` fixture (telegram_chat_id=None) to all endpoint tests
- **Flagged by**: Architect, PM, User, Security
- **Impact**: Core product actually works for web signups

#### 1.2 SEBI Compliance Decision (Week 1-2)
- **What**: Get formal legal opinion on SEBI RA registration requirement
- **Options**: (a) Register as RA (~₹10L + compliance), (b) Restructure signals as "educational analysis" — remove explicit BUY/SELL labels and target/stop-loss from NSE stock signals, (c) Focus on crypto/forex only
- **Immediate**: Disable signal sharing public URLs (`/shared/{id}`) — highest regulatory risk feature
- **Immediate**: Add explicit forex FEMA warning
- **Flagged by**: Compliance, Investor, PM

#### 1.3 Fix Fake Candlestick Data (Week 1)
- **What**: Replace `Math.random()` OHLC generation with real data or remove candlestick chart
- **File**: `frontend/src/app/signal/[id]/page.tsx` lines 207-218
- **Fix**: Either fetch real OHLCV from backend or default to line chart (uses real close prices)
- **Flagged by**: UX Lead, User — trust-destroying for finance professionals

#### 1.4 Database Backups + Retention (Week 2)
- **What**: Automated `pg_dump` daily backup + TimescaleDB retention policy
- **Policy**: `add_retention_policy('market_data', INTERVAL '90 days')`
- **Flagged by**: DevOps, Architect, Security

#### 1.5 Reduce Sentiment Weight to 15% (Week 2)
- **What**: Change scorer from 60/40 (tech/sentiment) to 85/15 until calibration data exists
- **File**: `backend/app/services/signal_gen/scorer.py`
- **Set**: `temperature=0` on all Claude scoring calls
- **Use**: Claude `tool_use` for structured JSON output
- **Flagged by**: Data Scientist, Quant — uncalibrated 40% weight is "reckless"

#### 1.6 Celery Task Retries (Week 2-3)
- **What**: Add `autoretry_for`, `retry_backoff=True`, `max_retries=3` to all 12+ tasks
- **Files**: All files in `backend/app/tasks/`
- **Flagged by**: Architect, DevOps — silent data gaps

#### 1.7 Process Supervision on Railway (Week 3)
- **What**: Replace bash `&` backgrounding with `supervisord` or split into separate services
- **File**: `railway.toml`, `start.sh`
- **Flagged by**: DevOps — Celery can silently die

### Phase 2: SECURITY HARDENING (Weeks 3-5)

#### 2.1 Server-Side Tier Enforcement (Week 3-4)
- **What**: Add `Depends(require_tier("pro"))` to backtest, AI Q&A, advanced endpoints
- **File**: `backend/app/auth.py` (already defined, never used), all `api/*.py` routes
- **Flagged by**: Security, PM, Investor

#### 2.2 Scope API Keys (Week 4)
- **What**: Separate internal-only API key from user-facing auth
- **Fix**: Never accept API key auth on user-facing endpoints
- **Create**: Per-service API keys with restricted scopes
- **Flagged by**: Security — single key = total compromise

#### 2.3 WebSocket Credential Security (Week 4)
- **What**: Replace JWT-in-query-params with one-time ticket tokens
- **Pattern**: `POST /ws/ticket` → returns single-use ticket → connect with ticket
- **Flagged by**: Security — credentials in URLs leak via logs/history

#### 2.4 CSP Fix (Week 4)
- **What**: Remove `'unsafe-inline' 'unsafe-eval'` from CSP, use nonce-based script loading
- **File**: `backend/app/main.py` line 101
- **Flagged by**: Security — XSS protection nullified

#### 2.5 JWT Revocation Store (Week 5)
- **What**: Redis-backed JTI blacklist, checked on every token validation
- **Trigger**: Logout, password change, account deletion add JTI to blacklist
- **TTL**: Match access token expiry (30 min)
- **Flagged by**: Security, Architect

#### 2.6 Dependency Lockfile (Week 5)
- **What**: Generate `requirements.lock` or switch to `pip-tools` / `poetry.lock`
- **Replace**: `psycopg2-binary` with `psycopg2` in production
- **Flagged by**: Security — supply-chain attack risk

### Phase 3: SIGNAL QUALITY (Weeks 5-8)

#### 3.1 Multi-Timeframe Analysis (Week 5-6)
- **What**: Run indicators on daily + weekly (stocks) or 4H + daily (crypto)
- **Rule**: Only generate signals when both timeframes confirm
- **File**: `backend/app/services/signal_gen/generator.py` — `_fetch_market_data()`
- **Flagged by**: Quant, Day Trader, Data Scientist — "single timeframe is unreliable"

#### 3.2 Market Regime Filter (Week 6-7)
- **What**: Add ADX(14) indicator; suppress trend signals when ADX < 20, suppress counter-trend when ADX > 40
- **File**: `backend/app/services/analysis/indicators.py`
- **Flagged by**: Quant — "will dramatically reduce whipsaw losses"

#### 3.3 Confidence Calibration (Week 7-8)
- **What**: After 200+ resolved signals, fit Platt scaling (logistic regression) to map raw scores to actual win probability
- **Display**: "65% historical accuracy" instead of "92% confidence"
- **Formula**: $\text{calibration}(s) = P(\Delta p > 0 | \text{score} = s)$
- **Flagged by**: Quant, Data Scientist — "dangerously misleading for learning traders"

#### 3.4 Backtest Parity with Live Pipeline (Week 8)
- **What**: Rewrite backtest to use actual `TechnicalAnalyzer.full_analysis()` + `compute_final_confidence()`
- **Method**: Walk-forward validation — train months 1-6, test month 7, slide forward
- **Current problem**: Backtest uses simplified RSI+SMA20 strategy, not the real pipeline
- **Flagged by**: Quant, Investor — "zero predictive validity for live system"

#### 3.5 Portfolio-Level Risk Controls (Week 7-8)
- **What**: Before emitting signal, check sector concentration, aggregate exposure, correlation
- **Rule**: `max_concurrent_per_sector = 2`, position sizing = `max_risk / (stop_distance × lot_size)`
- **Flagged by**: Quant, User — "5 banking stocks at once = hazardous"
